وجيه تعديل التطبيق (Replit)
المطلوب

عند فتح صفحة التمرين (/exercise/transformation) أو تبديل اللغة من زر اللغات:

تُعرَض الحِكمة/المثل باللغة المحدّدة حاليًا في واجهة المستخدم (en, ar, id, tr, …).

لا تُعرَض النسخة العربية إلّا إذا كانت اللغة المختارة “Arabic”.

كل النصوص الثابتة في البطاقة (العناوين، الأزرار، placeholder) تتبع اللغة نفسها.

أين نلمس الشفرة

Front-End: مكوّن صفحة التمرين (مثلاً ExerciseTransformation.tsx أو ما يقابله) + مكوّن زر اللغات/المترجم (Language Switcher).

Back-End / API: مسار جلب بيانات التمرين (مثلاً /api/exercise/transformation) ومسار الخدمة المساعدة للترجمة (جديد: /api/translate أو داخل نفس المسار كخيار).

تغييرات الواجهة (Front-End)

مصدر اللغة الحالية

اجلب اللغة من نظام الترجمة لديك (react-i18next/next-i18next أو الحالة العالمية).

لنسمّيها currentLang (قيم محتملة: en, ar, id, tr, …).

جلب الحكمة مترجمة

عند التحميل أو عند تغيير currentLang:

استدعِ API مع بارام targetLang=currentLang وwisdomId (أو النص نفسه إذا لا يوجد ID).

استبدل العنوان “Arabic Wisdom:” بـ “Wisdom:” فقط، واعرض النص المترجَم.

سلوك التبديل

عند تغيير اللغة من زر اللغات:

حدّث currentLang → أعد الجلب → حدّث عرض الحكمة دون إعادة تحميل الصفحة.

نصوص الواجهة

اربط نصوص: Exercise, Write a word…, Check Answer, Try Again, About This Phrase, Type, Difficulty… بملفّات i18n.

ضَع Placeholders لكل لغة (بما فيها العربية).

الاتجاه (RTL/LTR)

إذا currentLang === 'ar' → dir="rtl" + محاذاة يمين.

غير ذلك dir="ltr".

تغييرات الخادم (Back-End / API)

مدخل التمرين

أضف قبول بارام targetLang.

أعدَ إرسال الحِكمة مترجمة إن كانت targetLang !== 'ar'، وإلا أعد النسخة العربية الأصلية.

إن وُجد wisdomId:

جرّب جلب ترجمة مخزّنة من Cache/DB، وإن لم توجد ترجم الآن ثم خزّن النتيجة.

خدمة الترجمة

دالة translateText(text, targetLang):

مفضّل: Gemini API (الذي تستخدمه سلفًا) بصيغة prompt واضحة:

“Translate the following aphorism into <LANG_CODE>, keep it concise and natural, no extra commentary:”

بديل: Google Cloud Translate / DeepL.

أعد النتيجة كنص نقي بدون وسوم.

التخزين المؤقت (Cache)

جدول/ملف wisdom_translations بمفاتيح: (wisdom_id, target_lang) + النص المترجم + updated_at.

صلاحية: 30 يومًا (أعد الترجمة إذا انتهت أو أُغلِقت quota).

المعالجة الاستثنائية

Rate limiting / quota: في حال الفشل، أعِد آخر ترجمة مخزّنة؛ وإن لم توجد، اعرض الحِكمة بالعربية مع شارة “(ترجمة غير متاحة الآن)”.

أمان المخرجات: قصّ المسافات/الأسطر المضافة، امنع إدخال HTML.

تغيير قواعد العرض في البطاقة

العنوان: Wisdom (غير مقيّد بالعربية).

النص: محتوى الحِكمة باللغة الحالية فقط.

قسم “About This Phrase” و“Difficulty” و“Type” تُترجم مع i18n.

Placeholder خانة الإجابة يُترجم كذلك.

قبول المهمة (Acceptance Criteria)

عند اختيار English في زر اللغات، تظهر الحِكمة بالإنجليزية فقط.

عند اختيار Turkish، تظهر بالتركية فقط… وهكذا لباقي اللغات المفعّلة.

تبديل اللغة لحظي بدون تحديث الصفحة (hot swap).

تظهر العربية فقط عندما تكون اللغة المختارة Arabic.

النصوص الثابتة والأزرار والـ placeholders تتبع اللغة نفسها.

لا تظهر سطور عربية إضافية بجانب المترجم.

في انقطاع الترجمة: رسالة مهذّبة مع استمرار التمرين.

نقاط تنفيذ سريعة (Checklist)

 توحيد مصدر اللغة currentLang من زر اللغات.

 تمرير targetLang إلى API جلب التمرين.

 إضافة طبقة ترجمة + Cache بالـ Back-End.

 ربط جميع نصوص البطاقة بنظام i18n.

 دعم RTL عندما currentLang = 'ar'.

 اختبارات لغات: ar, en, id, tr (على الأقل).

 اختبار تبديل فوري للغة أثناء فتح الصفحة.

 مسار فشل واضح لوصول الترجمة (rate limit / quota).

ملاحظات خاصّة بالصورة/الوضع الحالي

نص “Arabic Wisdom:” ثابِت الآن — يجب استبداله بـ “Wisdom:” وربطه بمُخرَج الترجمة.

تأكّد أن زر اللغة أعلى يمين يغيّر currentLang في الحالة العامّة للتطبيق لا في واجهة واحدة فقط.

إن كانت الصفحة تستخدم SSR/ISR (Next.js)، مرّر targetLang عبر query أو header (مثل Accept-Language) وادعَم Client-Side re-fetch عند تغيير اللغة من الزر.
